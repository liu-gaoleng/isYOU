## 1. 产品定位（一分钟掌握）

- **一句话**：每天 10 分钟，读懂 **科技 / 金融 / AI / 宏观** 四个赛道最值得关注的事。
- **形态**：AI 聚合 + 提炼 + 轻人工质检的每日热点简报 App（目标形态为原生 App，iOS/Android，选型 Flutter）。
- **目标用户**：互联网从业者、产品经理、投资人、创业者。
  - 北极星人群 = 投资人 + 创业者（付费能力最强，决定营收上限）
  - 增长人群 = 产品经理 + 互联网从业者（决定 DAU 与口碑）
- **核心壁垒**：AI 内容引擎（采集→去重→分类→聚类→摘要→评分）+ 运营后台人工质检。
- **商业模式**：免费 / 会员两档订阅，付费点是「深度解读」（详情页付费墙）。报告商城、圈子为 v2 方向。
- **设计语言**：「终端·情报流」深色风格——深石墨底、琥珀金强调、涨跌红绿、等宽字体（详见原型 `:root` CSS 令牌）。

> 权威依据文档：[PRD.md](./PRD.md)（产品需求）、[MVP.md](./MVP.md)（范围与优先级）、[API设计.md](./API设计.md)、[内容管线方案.md](./内容管线方案.md)。本交接文档是上述文档的「当前进度视图」，遇冲突以四份原始文档为准。

## 4. 本地启动与验证

```bash
# 1. 启动 Mock 后端（终端）—— 端口 8000
python3 mock_server/server.py 8000

# 2. 启动原型静态服务 —— 端口 4173
python3 -m http.server 4173 --directory prototype

# 3. 浏览器访问
#    C 端：http://localhost:4173/index.html
#    后台：http://localhost:4173/admin.html

# 重启后端（改了 server.py 后必须重启，且清掉内存态脏数据）：
lsof -ti tcp:8000 | xargs kill -9 2>/dev/null; sleep 1; python3 mock_server/server.py 8000

# 真实跑内容管线 Demo（可选，会联网抓 RSS）：
cd pipeline_demo && pip3 install -r requirements.txt && python3 run_pipeline.py
# 启用 LLM 摘要（不设则走抽取式兜底）：export RD_LLM_API_KEY / RD_LLM_BASE_URL / RD_LLM_MODEL
```

### Mock 后端约定（接手必读）
- 统一响应包络：`{code, message, data, request_id}`，`code==0` 为成功。
- 会员态通过请求头模拟：`Authorization: Bearer member-token` → 会员（解锁付费墙/会员价）；空或其他 token → 游客/免费。
- 内存态管理（**重启即重置**）：FAVORITES / HISTORY / PUSH_SETTINGS / SOURCES / ADMIN_USERS / ROLE_PERMS / APP_USERS / REPORTS / PURCHASED_REPORTS。
- 端点清单在 `server.py` 的 `ENDPOINT_LIST`（line ~1118），访问 `/` 可列出。

## 6. 已知问题与历史决策（避免重复踩坑）

1. **宏观内容稀少**：output.json 中 macro 仅 4 条且重要度低，曾被排到首页尾部。
   - 决策：首页 `/v1/home/briefing` 改为**四模块均衡配额轮询混排**（科技→金融→AI→宏观循环填充），头条/热榜仍按 importance。宏观 4 条现稳定进首页（位置 4/8/12/16）。
2. **频道分页 size 不一致**：初版静态卡 + 动态卡 size 不匹配导致丢项/跳页。
   - 决策：统一 size=6，前 2 条填静态卡，第 3 条起用 `feedCardHtml` 追加，`loadMoreChannel` 用相同 size。
3. **静态卡与动态卡双重绑定**：静态 `.card` 加 `el._bound=true` 标记，`bindChannelCards` 检查后跳过。
4. **Mock 内存态脏数据**：购买报告、收藏等会写内存态。**端到端验证后务必重启 Mock 清理**（如测试购买了 rpt_1，重启后恢复未购态）。预置已购为 `PURCHASED_REPORTS={"guest":["rpt_2","rpt_3"]}`。
5. **server.py do_GET 变量约定**：查询参数变量名是 `q`（不是 `query`）；`tier` 已被会员态占用，勿复用。

## 7. 前端原型代码导航（index.html）

- 全局：`var TOKEN`（''=游客 / 'member-token'=会员）、`api()/apiPost()` 读 TOKEN 设 Authorization 头、`MODULE_CN` 模块中文名。
- 首页：`loadHome()`；频道：`loadChannel(phone)` / `loadMoreChannel()` / `feedCardHtml()`。
- 搜索：`openSearch()/runSearch()/fetchSearch()`，弹层 `#searchModal`。
- 详情：`openEvent()/closeEvent()`，弹层 `#modal` + 付费墙。
- 我的：`loadMe()`、`openFav()/openHistory()/openSettings()`，共用弹层 `#meModal`。
- 报告：`loadReports()/openReport()/buyReport()/openMyReports()`，弹层 `#rptModal`。
- 登录：`openLogin()/sendSms()/doLogin(type)/doLogout()`，弹层 `#loginModal`。
- 启动区在文件末尾：`loadHome(); loadMe(); loadReports(); 各频道 loadChannel`。

## 8. 建议的下一步（按 MVP 关键路径）

> MVP.md 明确：**M1 内容引擎必须先于 M2 客户端联调就绪**，否则 App 无内容可读。

1. **AI 内容引擎生产化**（最高优先，关键路径）：把 `pipeline_demo` 升级为可调度服务——定时采集、pgvector 去重聚类、LLM 分类/摘要 + 防幻觉护栏、产出入 PostgreSQL，并替换 Mock 数据源。
2. **后端服务化**：用 Go/FastAPI + PG + Redis 实现 API 设计.md 的契约，逐步替换 `mock_server`。
3. **支付闭环**（关键路径）：订单系统 + Apple IAP + 安卓支付 + 恢复购买 + 到期降级。
4. **Flutter 客户端**：按原型交互落地真实 App（简报/频道/详情/付费墙/我的/登录）。
5. **推送 + 埋点 + 监控**：APNs/FCM 早报、神策/GA4 漏斗、Sentry。

> 接手时建议：先 `python3 mock_server/server.py 8000` + 起原型，对照本文 §4 跑通现状；再读 MVP.md §2 功能清单逐项核对验收标准，从 §8 第 1 步切入。
