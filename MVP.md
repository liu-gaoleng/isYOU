# 「热读」MVP 功能清单与排期

> 配套文档：PRD.md（v1.1）｜ 原型：prototype/index.html
> 版本 v1.0 ｜ 状态：待评审 ｜ 形态：原生 App（iOS / Android，Flutter）
> 目标：定义首版**做什么 / 不做什么**，拆功能到可开发粒度，给出优先级、技术选型、验收标准与里程碑。

---

## 0. 阅读说明

### 优先级定义
| 级别 | 含义 | 处理原则 |
|---|---|---|
| **P0** | MVP 必须有，缺了无法上线 | 首版交付，不可砍 |
| **P1** | 重要但首版可简化/延后 | 上线后紧接迭代（v1.1–v1.2） |
| **P2** | 增强/商业化深水区 | 放到 v2+，本期不投入 |

### MVP 范围一句话
> **首版只做一件事：让目标用户每天打开 App，10 分钟读完科技/金融/AI/宏观四个赛道当天最值得看的事，并能为「深度解读」付费。**
> 一切不服务于这条主线的功能，本期一律不做。

---

## 1. MVP 边界：做什么 / 不做什么

### ✅ 首版做（P0）
- 今日简报首页（四模块聚合 + 今日热榜）
- 四大模块频道（科技/金融/AI/宏观）+ 各频道 TOP10 热榜
- 事件详情页（结构化摘要 + 多源 + 付费墙）
- AI 内容引擎（采集→去重→分类→聚类→摘要→评分）+ 运营后台人工质检
- 每日定时推送（早报）
- 注册登录 + 收藏 + 阅读历史
- 基础会员订阅（免费 / 会员两档）+ IAP 支付

### ❌ 首版不做（推到 v1.x / v2）
| 砍掉的功能 | 放到 | 理由 |
|---|---|---|
| 专业版会员、私董会优先 | v2 | 高净值运营，需内容与口碑沉淀后再上 |
| 数据看板（指数/融资图表） | v1.2 / v2 | 数据源与可视化成本高，非主线 |
| 付费报告商城 | v2 | 需先有稳定深度内容产能 |
| 圈子 / 轻社区 / 活动报名 | v2 | 社区是另一套产品，避免分散 |
| 数据 API / B 端定制 | v2+ | 商业化深水区 |
| 晚报推送、个性化推荐排序 | v1.1 | 先验证早报与统一排序 |
| 评论 / 点赞 / UGC | v2 | 非核心，先保内容质量 |

> 原型中已画出的「报告、圈子」页面属于 v2 视觉预研，**本期不开发**，仅保留 Tab 占位或隐藏。

## 2. 功能清单（按模块拆，含优先级 / 技术选型 / 验收标准）

### 2.1 今日简报（首页）

| # | 功能点 | 优先级 | 技术选型 / 实现 | 验收标准 |
|---|---|---|---|---|
| 1.1 | 日更卡片流（四模块聚合，按重要度排序） | **P0** | Flutter `ListView` + 后端排序接口；下拉刷新、上拉加载 | 打开首页 2s 内出当日内容；按重要度降序；空/加载/失败态完整 |
| 1.2 | 今日热榜 TOP（首页聚合榜） | **P0** | Redis ZSet 排行榜，热度分实时写入 | 榜单 ≤ 10 条，含排名/标题/热度/趋势；点击进详情 |
| 1.3 | 顶部 slogan + 日期 + 当日条数 | **P0** | 静态文案 + 接口返回当日发布数 | 文案正确显示；条数与后端一致 |
| 1.4 | 市场情绪条（指数/比特币速览） | **P1** | 第三方行情 API（延迟行情即可）；缓存 60s | 数值与涨跌色正确；接口失败时整条隐藏不报错 |
| 1.5 | 查看完整榜单 / 加载更多 | **P0** | 分页接口 `page/size` | 点击加载下一页，无重复、无丢失 |

### 2.2 四大模块频道

| # | 功能点 | 优先级 | 技术选型 / 实现 | 验收标准 |
|---|---|---|---|---|
| 2.1 | 频道切换（科技/金融/AI/宏观） | **P0** | Flutter `TabBar` + 懒加载各频道 | 切换流畅无白屏；记住上次所在频道 |
| 2.2 | 各频道 TOP10 热榜 | **P0** | 按模块维度的 Redis 榜单 | 每频道独立 10 条；排名/趋势/热度正确 |
| 2.3 | 要闻解读卡片流 | **P0** | 按模块过滤的事件流接口 | 榜单下方展示该模块要闻卡；含「为何重要」 |
| 2.4 | 「更多 · 你可能也关注」+ 加载更多 | **P0** | 同模块延展推荐（先用同分类热度兜底） | 推荐条目可点击进详情；加载更多正常 |
| 2.5 | 今日数字速览（金融/宏观） | **P1** | 行情/经济数据接口 | 数值正确；无数据时模块隐藏 |
| 2.6 | 标签订阅（自选关注） | **P2** | 用户标签表 + 个性化排序 | （本期不做，v2） |

### 2.3 事件详情页

| # | 功能点 | 优先级 | 技术选型 / 实现 | 验收标准 |
|---|---|---|---|---|
| 3.1 | 结构化正文（导语/小标题/要点/引用/重点） | **P0** | 后端返回结构化字段，前端按 schema 渲染 | 排版层级清晰；正文对比度达标（可读性优先） |
| 3.2 | 多源标注（X 源交叉 + 原文链接） | **P0** | 事件聚类产出的来源列表 | 至少展示来源数与可点击原文链接 |
| 3.3 | 付费墙（免费截断 + 会员引导） | **P0** | 服务端判定会员态返回完整/截断内容 | 非会员只见免费段 + 付费墙；会员见全文 |
| 3.4 | 收藏 / 分享 | **P0** | 收藏入库；分享生成带水印卡片或链接 | 收藏即时生效并出现在「我的」；分享可拉起系统面板 |
| 3.5 | 免责声明（金融非投资建议） | **P0** | 详情页底部固定文案 | 金融/宏观类必显；合规可查 |

### 2.4 AI 内容引擎（核心壁垒，P0 全链路）

> 详细技术方案见后续《内容管线方案》文档；此处定义 MVP 必须跑通的最小链路与验收线。

| # | 阶段 | 优先级 | 技术选型 / 实现 | 验收标准 |
|---|---|---|---|---|
| 4.1 | 信源采集 | **P0** | RSS + 媒体 API + 第三方新闻聚合；定时任务 | 覆盖四模块种子信源 ≥ 40 个；定时无遗漏抓取 |
| 4.2 | 清洗去重 | **P0** | 正文抽取 + Embedding 相似度去重（pgvector） | 同事件多源合并；重复率 < 5% |
| 4.3 | 分类（四模块 + 标签） | **P0** | LLM 分类 + 规则兜底 | 分类准确率 ≥ 90%（抽检）；无法分类入待审 |
| 4.4 | 事件聚类 | **P0** | 向量聚类（相似度阈值 / HDBSCAN） | 同一事件聚为一卡，串卡率可接受（抽检） |
| 4.5 | AI 摘要 / 解读生成 | **P0** | LLM + Prompt 模板（事实/解读分区，强制引用） | 摘要含：标题/3句摘要/为何重要/来源；不杜撰数字 |
| 4.6 | 重要性评分与排序 | **P0** | 加权评分（来源权重·交叉数·热度·时效） | 评分驱动榜单与首页排序；可后台调权重 |
| 4.7 | 运营后台人工质检 | **P0** | Web CMS：审核/编辑/下架/置顶/推送 | 高优内容可人工抽检；一键修正与发布 |
| 4.8 | 防幻觉与合规护栏 | **P0** | 引用校验 + 敏感词 + 金融免责 | 缺原文不发布；事实与解读分区；合规通过 |

## 3. 技术选型汇总（MVP）

| 层 | 选型 | 说明 |
|---|---|---|
| 客户端 | **Flutter**（iOS + Android 一套码） | 团队小、迭代快，资讯类性能足够 |
| API 服务 | **Go**（或 Python FastAPI），单体起步 | 先单体后微服务，避免过早拆分 |
| 业务库 | **PostgreSQL** | 用户/内容/订单 |
| 缓存/榜单 | **Redis**（ZSet 做热榜） | 热度排行榜、会话缓存 |
| 向量库 | **pgvector**（起步）/ Milvus（规模化） | 去重、聚类、语义检索 |
| 对象存储 | 云 OSS / S3 | 图片、卡片图 |
| AI | 云 LLM API + 自部署开源模型（控成本） | 分类/摘要/解读；Embedding 做向量 |
| 调度 | **Celery / Airflow** | 采集与内容管线定时编排 |
| 推送 | APNs + FCM + 厂商通道 | 保证到达率 |
| 支付 | Apple IAP + 安卓支付 | 抽成计入定价 |
| 运营后台 | Web CMS（React/Vue 任一） | 审核、编辑、推送、用户/订单管理 |
| 监控 | Sentry + 神策/GA4 | 崩溃 + 业务分析 |

## 4. 里程碑排期（相对周次，T = 开发启动）

> 采用相对周次，不绑定具体日历。可并行的工作流分为 **内容引擎**、**客户端**、**后端/后台** 三条线。

### 阶段总览
| 阶段 | 周次 | 目标 | 关键交付 | 出口标准（Gate） |
|---|---|---|---|---|
| **M0 准备** | T+0 ~ T+2 | 拉通基础 | 技术架构定稿、设计系统、信源清单、脚手架 | 三端脚手架可跑通；CI 就绪 |
| **M1 内容引擎 MVP** | T+2 ~ T+6 | 跑通内容生产 | 采集→去重→分类→聚类→摘要→评分 + CMS 质检 | **内部灰度日更内容，抽检合格率 ≥ 90%** |
| **M2 客户端核心** | T+4 ~ T+9 | 跑通阅读闭环 | 简报/频道/热榜/详情/付费墙/账号/收藏 | 真机可读当日内容，主流程无阻断 |
| **M3 商业化 + 推送** | T+8 ~ T+11 | 跑通付费与触达 | 会员订阅 + IAP + 早报推送 + 埋点 | 支付闭环通过；推送到达；漏斗可查 |
| **M4 联调灰度** | T+11 ~ T+13 | 提质量 | 全链路联调、性能、合规、内测 | 崩溃率达标；合规过审准备就绪 |
| **M5 上线** | T+13 ~ T+15 | 上架 | 双端提审、灰度放量、监控值守 | 双端过审上线；核心指标可观测 |

> 关键依赖：**M1 内容引擎必须先于 M2 客户端联调就绪**——否则 App 无内容可读。M1 是全局关键路径，优先投人。

### 关键路径与风险缓冲
- **关键路径**：M1 内容引擎质量 → M2 详情页/付费墙 → M3 支付。
- **缓冲**：M4 预留 2 周做合规、性能、苹果审核往返（IAP 审核易反复）。
- **可并行**：M0 后端脚手架、客户端 UI、内容引擎三线并行，减少串行等待。

## 5. 上线前必过检查项（Release Checklist）

- [ ] 内容：当日四模块均有内容，抽检合格率 ≥ 90%，无幻觉/无杜撰数字
- [ ] 合规：金融内容免责声明齐全；摘要均挂原文链接；信源授权策略经法务确认
- [ ] 付费：IAP 闭环（购买/恢复/到期降级）双端验证通过
- [ ] 推送：早报定时到达，点击直达；可关闭
- [ ] 体验：冷启动 < 3s；崩溃率 < 0.5%；空/加载/失败态完整
- [ ] 设计：全局符合 §3.5「简洁高级（克制但有张力）」基调
- [ ] 数据：核心漏斗（启动→阅读→付费）埋点可查

## 6. MVP 成功判定（上线后 4–8 周看）

| 维度 | 指标 | 及格线（参考，需结合实际校准） |
|---|---|---|
| 内容质量 | AI 摘要人工抽检合格率 | ≥ 95% |
| 留存 | 次日留存 / 7 日留存 | 行业资讯类对标，持续优化 |
| 活跃 | 简报完读率、推送打开率 | 完读率为北极星辅助指标 |
| 商业 | 付费转化率、首充续费率 | 验证「深度解读」付费意愿是否成立 |

> **MVP 的首要验证目标**：内容质量是否过关 + 是否有人愿为深度解读付费。这两点成立，才进入 v2 的数据/报告/社群扩张。
