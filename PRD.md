## 1. 产品定位与目标用户

### 1.1 一句话定位
**"每天 10 分钟，读懂科技、金融、AI、宏观四个赛道最值得关注的事。"**
一款由 AI 聚合 + 提炼、为高知职业人群节省信息筛选成本的每日热点简报 App。

### 1.2 我们解决的核心问题
- **信息过载**：从业者每天要刷十几个公众号/媒体/推特，噪音大、重复多。
- **缺乏提炼**：原始新闻冗长，缺少"这条事和我有什么关系"的解读。
- **跨领域盲区**：产品经理想懂金融、投资人想懂 AI 技术，缺一个统一入口。
- **决策滞后**：宏观与行业信号分散，难以快速形成判断。

### 1.3 与现有产品的差异
| 对标 | 它们的问题 | 我们的差异 |
|---|---|---|
| 今日头条/各类资讯流 | 泛大众、算法娱乐化、信息密度低 | 垂直高知人群、强提炼、反信息茧房 |
| 公众号/Newsletter | 分散、更新不稳定、无结构 | 统一入口、每日定时、结构化简报 |
| 财经终端（Wind 等） | 贵、重、偏专业机构 | 轻量、日更、解读友好 |
| 即刻/雪球 | 社区噪音大 | 编辑级信息质量 + 轻社区 |

### 1.4 目标用户画像

| 画像 | 核心诉求 | 付费意愿 |
|---|---|---|
| **互联网从业者**（运营/市场/研发） | 快速掌握行业大事、不掉队 | 中 |
| **产品经理** | 看竞品、看趋势、找灵感 | 中高 |
| **投资人**（一二级） | 信号、深度解读、数据、闭门信息 | **高** |
| **创业者** | 赛道动态、政策、融资风向 | **高** |

> 北极星人群 = **投资人 + 创业者**（付费能力最强，决定营收上限）；
> 增长人群 = **产品经理 + 互联网从业者**（决定 DAU 与口碑传播）。

## 2. 核心壁垒：AI 内容引擎

内容质量是这类产品的生死线。我们采用 **AI 自动抓取 + 聚合摘要 + 轻人工质检** 的混合流水线。

### 2.1 内容生产流水线（Pipeline）

```
信源采集 → 清洗去重 → AI 分类（四大模块）→ 事件聚类 → AI 摘要/解读 → 质量分 → 人工质检（可选）→ 发布
```

| 阶段 | 做什么 | 技术手段 |
|---|---|---|
| **1. 采集** | 多源抓取：RSS、媒体 API、公众号、X/Twitter、研报、政府公告 | 定时爬虫 + 第三方聚合 API |
| **2. 清洗去重** | 去广告、去模板、正文抽取；同一事件多来源合并 | 正文抽取 + 向量相似度（embedding）去重 |
| **3. 分类** | 自动打到 科技/金融/AI/宏观 四模块 + 细分标签 | LLM 分类 + 规则兜底 |
| **4. 事件聚类** | 把"同一件事的 N 篇报道"聚成一个事件卡片 | 向量聚类（HDBSCAN/相似度阈值） |
| **5. 摘要/解读** | 生成：标题、3 句话摘要、为什么重要、影响谁 | LLM（带 Prompt 模板 + 引用原文防幻觉） |
| **6. 质量分** | 重要性打分（来源权威度 + 传播热度 + 时效） | 加权评分模型，决定排序与推送 |
| **7. 质检** | 高优先级/付费内容人工抽检，防错防幻觉 | 运营后台一键审核 |

### 2.2 信源分级（保证权威性）
- **S 级**（强权威）：官方公告、监管机构、头部财经媒体、公司财报。
- **A 级**：垂直专业媒体、知名分析师、券商研报。
- **B 级**：自媒体、社交平台热议（仅作热度信号，需交叉验证）。

> 重要性评分 = 来源等级权重 × 0.4 + 多源交叉数 × 0.3 + 传播热度 × 0.2 + 时效 × 0.1

### 2.3 防幻觉与合规（关键风险控制）
- **强制引用**：每条摘要必须挂原文链接与来源；AI 解读与事实分区展示。
- **不杜撰数据**：涉及数字/财务，必须来自原文，缺失则留空而非编造。
- **版权合规**：只做"摘要 + 链接回源"，不全文转载；商用前需法务确认信源授权策略。
- **金融合规**：所有解读标注"非投资建议"；不发布荐股等敏感内容。

## 3.5 设计原则：简洁高级（核心设计准则）

> **总纲：界面服务于内容，而非装饰内容。「热读」面向高知人群，视觉上要传递"克制、专业、值得信赖、高级感"，做减法而非加法。**

凡是新增页面、组件、视觉稿，必须同时满足以下规则：

### 3.5.1 留白与密度
- 大胆留白，单屏只突出一件核心事；信息分层用间距而非分割线。
- 卡片之间用足够的呼吸感（间距 ≥ 16pt），拒绝拥挤堆砌。

### 3.5.2 色彩克制
- **黑白灰为主色**（高级感来自克制），仅用 **一个品牌强调色**点缀（建议低饱和的克制色，如墨绿/深蓝）。
- 四大模块用极淡的标签色区分，不大面积铺色；杜绝彩虹色、渐变滥用。
- 重要度等级用单色深浅表达，不用红黄绿"信号灯"式廉价配色。

### 3.5.3 字体与排版
- 一套字体家族 + 严格的字号层级（标题/正文/辅助 3–4 级），不混用多种字体。
- 中文正文行高宽松（1.6–1.8）；标题用更粗字重拉开层次，营造杂志/简报质感。

### 3.5.4 极简交互
- 一屏一焦点；减少按钮数量，次要操作收进二级。
- 去除多余图标与边框，多用文字与留白引导；动效轻、快、有目的（≤ 250ms）。

### 3.5.5 一致性与品质细节
- 统一圆角、统一阴影（弱阴影或无阴影）、统一图标线条粗细。
- 图片/Logo 留白对齐网格；空状态、加载态也要精致，不糊弄。

### 3.5.6 反面清单（禁止项）
- ❌ 花哨渐变、霓虹色、强对比撞色
- ❌ 信息流拥挤、卡片塞满、分割线密集
- ❌ 多字体混排、字号随意
- ❌ 弹窗轰炸、夸张动效、闪烁红点诱导
- ❌ 廉价感的强促销 banner（会员引导也要克制、高级）

> 一句话验收标准：**"像一份精心排版的高端简报/杂志，而不是一个塞满资讯的门户。"**

## 4. 四大模块详细设计

| 模块 | 覆盖内容 | 细分标签示例 | 核心用户 |
|---|---|---|---|
| **科技** | 大厂动态、新品、出海、监管 | 硬件/出海/平台/政策 | 互联网从业者、PM |
| **金融** | 市场、上市、并购、融资 | A股/美股/一级市场/加密 | 投资人 |
| **AI** | 模型、论文、产品、算力、应用 | 大模型/Agent/芯片/应用层 | 全体 |
| **宏观** | 政策、经济数据、地缘、行业 | 货币/财政/地产/出口 | 投资人、创业者 |

每个模块包含：
- **Top 事件流**（按重要度排序）
- **今日数字**（关键数据速览，如指数、融资额、模型榜单）
- **深度解读位**（会员专享，1–2 篇/日/模块）
- **标签订阅**（用户自选关注，影响推送与排序）

## 5. 商业模式与定价

三条变现线，按上线顺序推进：**会员订阅（主） → 付费报告/数据（高客单） → 社群/活动（高净值运营）**。

### 5.1 会员订阅（核心营收）

| 版本 | 价格（建议） | 权益 |
|---|---|---|
| **免费版** | ¥0 | 今日简报 · 四模块基础事件卡片 · 每日推送 |
| **会员版** | ¥30/月 · ¥298/年 | 全部深度解读 · 数据看板 · 历史检索 · 无广告 · 报告 8 折 |
| **专业版** | ¥98/月 · ¥980/年 | 会员全部 + 闭门简报 + 数据导出 + 私董会优先 |

> 免费看"是什么"，付费看"为什么 + 怎么办 + 数据"。

### 5.2 付费报告 / 数据服务（高客单价）
- 行业深度报告、赛道地图、融资数据库（单份 ¥99–¥999，会员折扣）。
- 面向机构的数据 API / 定制研究（B 端，按年签）。

### 5.3 社群 / 活动（高净值运营 + 品牌）
- 付费私董会、线下沙龙、闭门分享（¥1000+/场或年费制）。
- 品牌专栏 / 行业峰会合作（B 端赞助）。

### 5.4 收入结构预期（成熟期参考）
会员订阅 ~50% ｜ 付费报告/数据 ~30% ｜ 社群活动/B 端 ~20%

## 6. 技术选型

### 6.1 客户端（原生 App）
| 方案 | 说明 | 建议 |
|---|---|---|
| **Flutter** | 一套代码双端，迭代快，性能足够资讯类 | ✅ **推荐**（早期团队小、迭代快） |
| React Native | 生态成熟，前端友好 | 备选 |
| 纯原生 Swift/Kotlin | 体验最佳但成本翻倍 | 资源充足后局部优化 |

### 6.2 后端
- **API 服务**：Go / Python（FastAPI），微服务或单体起步。
- **数据库**：PostgreSQL（业务）+ Redis（缓存/排行榜）+ 向量库（Milvus/pgvector，做去重与聚类）。
- **对象存储**：图片/报告 PDF。

### 6.3 AI 内容引擎
- **采集**：Scrapy / 自研爬虫 + 第三方新闻 API + RSS。
- **LLM**：分类/摘要/解读用大模型（可混用云端 API + 自部署开源模型控成本）。
- **Embedding + 向量检索**：做去重、聚类、语义搜索。
- **任务调度**：定时管线（Airflow / Celery）。

### 6.4 基础设施
- 推送：APNs + FCM + 厂商通道（保证到达率）。
- 支付：App 内购（IAP，Apple 抽成需计入定价）+ 安卓应用市场支付。
- 后台：运营 CMS（审核、人工编辑、推送管理、用户/订单管理）。
- 埋点：自建或第三方（神策/GA4）做留存与转化分析。

## 7. MVP 范围与上线路线图

### 7.1 MVP（首版必须有 / 砍掉的）
**必须有：**
- 今日简报首页（聚合卡片流）
- 四大模块频道
- AI 内容引擎（采集→去重→分类→摘要）+ 运营后台人工质检
- 每日定时推送（早报）
- 注册登录 + 收藏
- 基础会员订阅（免费/会员两档）+ IAP 支付

**首版砍掉（放到 v2）：**
- 数据看板、付费报告商城、圈子社群、专业版、私董会、数据 API。

### 7.2 路线图
| 阶段 | 目标 | 主要交付 |
|---|---|---|
| **P0 内容引擎就绪**（上线前） | 跑通并验证 AI 内容质量 | 内容管线打通 + 运营后台质检，内部灰度跑通日更内容 |
| **P1 MVP 上线** | 跑通核心闭环 | 双端 App + 简报 + 四模块 + 推送 + 基础会员 |
| **P2 商业化增强** | 提升付费 | 深度解读、数据看板、付费报告商城、专业版 |
| **P3 社群与 B 端** | 拉高客单与壁垒 | 圈子/私董会/活动、数据 API、机构定制 |

> 决策：**不做小程序验证，直接投入原生 App 开发。** P0 阶段先把 AI 内容引擎跑通并通过内部灰度验证内容质量，确保 App 上线即有稳定、高质量的日更内容。

## 8. 关键指标与风险

### 8.1 北极星与关键指标
- **北极星**：付费会员数（及付费续费率）。
- 内容侧：日发布事件数、AI 摘要准确率（人工抽检合格率 ≥ 95%）、去重率。
- 增长侧：次日/7 日留存、推送打开率、简报完读率。
- 商业侧：付费转化率、ARPU、续费率、报告/活动 GMV。

### 8.2 主要风险与应对
| 风险 | 应对 |
|---|---|
| **AI 幻觉/错误信息** | 强制引用原文、事实/解读分区、高优内容人工质检 |
| **版权/合规** | 摘要+回源、法务确认信源授权、金融内容免责声明 |
| **内容同质化** | 强化"解读+数据+跨领域"差异，建立 S 级独家信源/专家网络 |
| **冷启动没人看** | P0 用 Newsletter/小程序低成本验证；绑定 KOL/社群分发 |
| **苹果 IAP 抽成** | 定价计入抽成；引导年费/官网/B 端绕开高抽成 |
