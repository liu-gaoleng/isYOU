## 7. 上线前必过检查项（Release Checklist，摘自 MVP.md §5）

- [ ] 内容：当日四模块均有内容，抽检合格率 ≥90%，无幻觉/无杜撰数字
- [ ] 合规：金融免责声明齐全；摘要均挂原文链接；信源授权策略经法务确认
- [ ] 付费：IAP 闭环（购买/恢复/到期降级）验证通过
- [ ] 推送：早报定时到达，点击直达；可关闭
- [ ] 体验：冷启动 <3s；崩溃率 <0.5%；空/加载/失败态完整
- [ ] 设计：全局符合「简洁高级（克制但有张力）」基调
- [ ] 数据：核心漏斗（启动→阅读→付费）埋点可查

## 8. 关键路径与建议起步顺序

1. **M1 收口**（§1.1）：保持 beat 常驻，3 天后抽检收口——不阻塞，可与 M2 并行。
2. **M2 iOS 工程脚手架 + 后端账号 API**（§2.1 / §3.1）：先打通「能登录、能读当日内容」的最小闭环。
3. **付费闭环**（§3.2/3.3 + §4.1）：账号就绪后做会员态 + IAP，这是商业化关键路径。
4. **推送 + 埋点**（§4.2/4.3）。
5. **联调 / 合规 / 部署 / 提审**（§5 / §6）。

> 铁律：M1 内容引擎须先于 M2 客户端联调就绪（已满足）；客户端与支付是当前最重的两块。
